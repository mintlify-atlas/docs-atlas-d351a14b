title: "Scientific Mode"
description: "Use trigonometry, logarithms, roots, factorials, and mathematical constants in OpenCalc's scientific mode."
---

Scientific mode adds two rows of advanced function buttons above the standard keypad. All scientific functions are evaluated using `BigDecimal` arithmetic for high precision, and results respect the configured decimal precision from Settings.

---

## Enabling scientific mode

In portrait orientation, scientific mode has three states controlled from **Settings ‚Üí Scientific mode**:

<Tabs>
  <Tab title="Toggle (default)">
    A small arrow button appears at the top of the keypad. Tap it to **show** or **hide** the scientific rows at any time. This is the default state ‚ÄĒ the toggle button is visible and the rows start collapsed.
  </Tab>
  <Tab title="Always open">
    The scientific rows are expanded and the toggle button is visible, but the rows are pre-expanded when you open the app.
  </Tab>
  <Tab title="Hidden">
    The toggle button and all scientific rows are removed. The app shows only the standard calculator layout.
  </Tab>
</Tabs>

<Note>
  In **landscape orientation**, the scientific keys are always visible regardless of this setting. Rotating your device is the quickest way to access scientific functions without changing settings.
</Note>

## Degree and radian mode

Trigonometric functions operate in **degrees by default**. The current mode is shown as a label next to the degree/radian toggle button.

- A text label ("Degree" or "Radian") shows the **current active mode**.
- The toggle button next to it shows the **other mode** ‚ÄĒ tap it to switch. The label and live preview update immediately.

To make radians the default when the app starts, enable **Use radians by default** in Settings.

## INV ‚ÄĒ inverse functions

Tapping **INV** switches the scientific function buttons to their inverses. Tap INV again to return to normal. The substitutions are:

| Normal | INV variant |
|---|---|
| `sin` | `sin‚ĀĽ¬Ļ` (arcsin) |
| `cos` | `cos‚ĀĽ¬Ļ` (arccos) |
| `tan` | `tan‚ĀĽ¬Ļ` (arctan) |
| `ln` | `exp` (eň£) |
| `log` | `10^` |
| `log‚āā` | `2^` |
| `‚ąö` | `x¬≤` (or `#` modulo if the modulo button is enabled in Settings) |

<Note>
  INV does not toggle degree/radian mode. Use the separate degree/radian button for that.
</Note>

## Error conditions

Scientific functions surface errors inline in the result display. The input text turns red to indicate a problem:

<AccordionGroup>
  <Accordion title="Domain error">
    Returned when the argument is outside the function's valid domain ‚ÄĒ for example, `ln(0)`, `sqrt(-1)`, or `arcsin(2)`.
  </Accordion>
  <Accordion title="Requires real number">
    Returned when a negative base is raised to a fractional power, such as `(-8)^0.5`, which has no real solution.
  </Accordion>
  <Accordion title="Value too large / Infinity">
    Returned when a result exceeds representable range ‚ÄĒ for example, `factorial(3000)` or very large exponents.
  </Accordion>
</AccordionGroup>
