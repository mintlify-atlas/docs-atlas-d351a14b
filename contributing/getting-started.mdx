OpenCalc is an open-source Android calculator built by the community. Whether you're fixing a bug, adding a feature, or improving tests, contributions are welcome and appreciated.

## Prerequisites

Before you begin, make sure you have the following installed:

- **Android Studio** (latest stable release recommended)
- **JDK 8** or higher (`sourceCompatibility = JavaVersion.VERSION_1_8`)
- **Android SDK** with API level 21 (Android 5.0) or higher

## Setting up your environment

<Steps>
  <Step title="Fork the repository on GitHub">
    Go to [github.com/Darkempire78/OpenCalc](https://github.com/Darkempire78/OpenCalc) and click **Fork** to create your own copy of the repository.
  </Step>
  <Step title="Clone your fork">
    ```bash
    git clone https://github.com/YOUR_USERNAME/OpenCalc.git
    ```

    Replace `YOUR_USERNAME` with your GitHub username.
  </Step>
  <Step title="Open the project in Android Studio">
    Open Android Studio, select **Open**, and navigate to the cloned `OpenCalc` directory. Android Studio will detect the Gradle project automatically.
  </Step>
  <Step title="Wait for Gradle sync to complete">
    Android Studio will download dependencies and sync the project. This may take a few minutes on first run. You can monitor progress in the **Build** tool window at the bottom.

    The project uses the following repository sources configured in `settings.gradle.kts`:
    - Google Maven
    - Maven Central
    - JitPack
  </Step>
  <Step title="Run on an emulator or physical device">
    OpenCalc requires **minSdk 21 (Android 5.0+)**. Set up an AVD in Android Studio or connect a physical device with USB debugging enabled, then click **Run**.
  </Step>
</Steps>

## Build commands

You can also build and test from the command line using the Gradle wrapper.

<Tabs>
  <Tab title="Debug build">
    ```bash
    ./gradlew assembleDebug
    ```

    The debug APK is output to `app/build/outputs/apk/debug/`. Note that debug builds use the application ID `com.darkempire78.opencalculator.debug`, so they can coexist with a release install.
  </Tab>
  <Tab title="Release build">
    ```bash
    ./gradlew assembleRelease
    ```

    Release builds have minification and resource shrinking enabled via ProGuard. The output APK is in `app/build/outputs/apk/release/`.
  </Tab>
  <Tab title="Run tests">
    ```bash
    ./gradlew test
    ```

    This runs the local JVM unit tests. To run instrumented Android tests on a connected device or emulator, use:

    ```bash
    ./gradlew connectedAndroidTest
    ```
  </Tab>
</Tabs>

## Code style

- All source code is written in **Kotlin**.
- The project follows standard Android and Kotlin conventions (4-space indentation, `camelCase` for variables, `PascalCase` for classes).
- View binding is enabled — use generated binding classes rather than `findViewById`.
- Keep functions focused and well-named. The calculator engine in particular (`Calculator.kt`) uses a recursive descent pattern; follow the existing style when adding parser changes.

## Pull request guidelines

<Info>
  For major changes, please **open an issue first** to discuss what you would like to change. This avoids duplicated effort and ensures alignment with the project's direction.
</Info>

- Keep your branch up to date with `main` before opening a PR.
- Update or add tests wherever appropriate — run `./gradlew test` to confirm they pass.
- Write a clear PR description explaining *what* changed and *why*.
- Reference any related issue numbers in the PR description (e.g., `Closes #123`).
- Keep PRs focused: one logical change per PR is much easier to review.
